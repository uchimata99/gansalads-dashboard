# שליחת וואטסאפ אוטומטית לספקים — מדריך הקמה

השולח הוא **Google Apps Script** (`wa_send.gs`) שרץ בגוגל, קורא את תור היציאה
(`WA_OUTBOX`) מגיליון הרכש ושולח כל הזמנה דרך **WhatsApp Cloud API** של מטא.
האפליקציה (`purchasing.html`) רק כותבת לתור — היא לא שולחת בעצמה.

**הסוד (הטוקן של מטא) חי אך ורק ב-Script Properties של הסקריפט — לעולם לא בקוד הציבורי.**

---

## שלב 1 — צד מטא (WhatsApp Business Platform)
מבצעת מיכל (או מי שמנהל את חשבון העסק):

1. **חשבון עסקי + אימות** ב-`business.facebook.com`.
2. אפליקציה ב-`developers.facebook.com` → מוצר **WhatsApp**.
3. **רישום המספר העסקי הייעודי** לממשק (מקבל קוד אימות חד-פעמי פעם אחת).
4. **אמצעי תשלום** על החשבון (חיוב פר-הודעה, קטגוריית Utility — אגורות).
5. **תבנית הודעה** בקטגוריית **Utility**, שפה `he`, עם שלושה משתני body בסדר:
   `{{1}}` שם הספק · `{{2}}` טקסט ההזמנה · `{{3}}` תאריך.
   נוסח מוצע לגוף התבנית:
   ```
   שלום {{1}},
   הזמנת רכש מסלטי גן:
   {{2}}
   תאריך: {{3}}
   תודה!
   ```
   מגישים לאישור (1–2 ימים).
6. בסוף אוספים שלושה ערכים: **מזהה מספר הטלפון** (Phone Number ID),
   **טוקן קבוע** (System User token), ו**שם התבנית** המאושרת.

## שלב 2 — הקמת הסקריפט
1. `script.google.com` → New project. להדביק את תוכן `wa_send.gs`.
2. **Project Settings ▸ Script Properties** — להוסיף:
   | מפתח | ערך |
   |------|-----|
   | `SHEET_ID` | `1rWHMhO8zCB8KKzAJwyFYpuKfo_EQ_-rZB8afaiqUv9Q` |
   | `META_TOKEN` | הטוקן הקבוע ממטא |
   | `PHONE_NUMBER_ID` | מזהה מספר הטלפון |
   | `TEMPLATE_NAME` | שם התבנית (למשל `purchase_order`) |
   | `TEMPLATE_LANG` | `he` |
   | `REGISTER_PIN` | PIN בן 6 ספרות לרישום המספר (רק לרישום החד-פעמי) |
3. **רישום המספר (חד-פעמי, עוקף את כפתור ה-Register התקול):** לבחור בעורך את הפונקציה
   `registerNumber` וללחוץ Run. לאשר הרשאות. לבדוק ב-Execution log שחזר `200`.
   המספר יעבור מ-`Pending` ל-`Connected`. (אם השם עדיין בבדיקה — זה בסדר, המספר עובד.)
4. הרצה ראשונה של `sendPendingOrders` ידנית — לאשר הרשאות (גישה לגיליון + רשת).
4. **Triggers** (⏰) → Add trigger → `sendPendingOrders`, Time-driven, Every minute
   (או כל 5 דקות).

## שלב 3 — הדלקת הכפתור באפליקציה
ב-`purchasing.html`, בראש הסקריפט, לשנות:
```js
const WA_AUTO=false;   →   const WA_AUTO=true;
```
commit + push + merge ל-`main`. מאותו רגע מופיע בסל כפתור **"🤖 שלח אוטומטית"**
לכל ספק: לחיצה מכניסה את ההזמנה לתור, והסקריפט שולח תוך דקה. הסטטוס נכתב
בחזרה ל-`WA_OUTBOX` (`sent`/`failed` + מזהה הודעה).

### הערות
- הכפתור הידני **"📲 שלח בוואטסאפ"** (פתיחת וואטסאפ במכשיר) נשאר תמיד כגיבוי.
- ספק בלי מספר בספר אנשי הקשר / בפנקס המכשיר → אי אפשר לשלוח אוטומטית
  (הכפתור מתריע). מוסיפים מספר דרך `ingest_contacts.py` או "☎️ מספר".
- מטא דורשת שהנמען יסכים לקבל הודעות (opt-in); לספקים עסקיים שמצפים להזמנה
  זה בדרך כלל תקין, אך כדאי לוודא מול הספקים.
